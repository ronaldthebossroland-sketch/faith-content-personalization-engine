Beginner Guide

A plain-English guide for using the dashboard, backend, and tracking tools.

🧭 What This Website Is

This is an admin dashboard for a faith-content personalization engine. It shows which approved apps are sending user activity, and turns that activity into anonymous profiles and content recommendations.

Vercel Dashboard The website you are looking at right now.
Render API The backend that receives tracking events.
Supabase Database Where users, consent, and events are stored.

🔗 Your Live URLs

These are the two main links in this setup:

Dashboard:
https://faith-content-personalization-engin.vercel.app

Backend API:
https://faith-content-personalization-engine.onrender.com

The dashboard should use the backend API URL in the API Endpoint field.

🖥️ How To Use The Dashboard

  1. 1Open the dashboard URL.
  2. 2Make sure API Endpoint is set to your Render backend URL.
  3. 3Enter your private ADMIN_API_KEY in the Admin Key field.
  4. 4Click Load Stats.
  5. 5The locked source cards will show real users, events, and profile summaries.

Do not share your admin key with normal website visitors. It is only for you or trusted admins.

📊 What The Dashboard Numbers Mean

Connected Apps Approved sources that are allowed to send activity to the engine.
Total Users Anonymous user profiles created by your app or tests — no real names stored.
Total Events Tracked actions like viewing content, reading articles, or watching videos.
Data Retention How many days the system keeps event data before automatically removing old records.

🧪 How To Test Without Another Website

  1. 1Scroll to Developer Tools and expand it.
  2. 2Click Create User to generate an anonymous profile.
  3. 3Click Accept Consent so the user is allowed to be tracked.
  4. 4Choose an event type, source, and topic.
  5. 5Click Track Event.
  6. 6Click View Profile to see the generated interest profile.
  7. 7Go back to the top and click Load Stats to see the new user in the admin cards.

🪪 Known User Mode

By default, this system only uses anonymous IDs like anon_.... Known User Mode lets a trusted admin connect an anonymous profile to a real person's name or email — but only after that person has logged in and given explicit consent.

Use this only when you have a clear reason to know who the person is, such as helping a logged-in member receive better recommendations.

How to link a known user in the dashboard

  1. 1Create or find the anonymous user ID in Developer Tools.
  2. 2Make sure the user has accepted consent.
  3. 3Open Developer Tools → Admin Tools.
  4. 4Fill in the person's name, email, or external user ID.
  5. 5Click Link Known User.
  6. 6Click Load Stats to see the name appear on admin profile cards.

⚠ Important: never expose your admin key

Do not put your admin key inside a public website or mobile app. If a real app needs to link names, its backend server should call the protected admin endpoint after login and consent. Normal visitor tracking should stay anonymous.

🌐 How To Connect A Real Website

Scroll to Connect Your App or Website on the dashboard. Choose the source for that website, then copy the embed script.

<script src="https://faith-content-personalization-engine.onrender.com/embed.js?source=official_ministry_website"></script>

Paste that script into the website you want to track. The script shows a consent banner first. If the user allows personalization, it starts sending approved activity to the backend automatically.

✅ What Counts As An Approved Source

An approved source is a website, app, page, or campaign you trust to send activity into this system.

healing_school_app healing_streams_registration_page approved_healing_school_campaign official_ministry_website approved_email_campaign

🔒 Privacy Rules

This system is designed to track only consented content activity. It should not collect:

  • Private messages or passwords
  • Bank details or payment information
  • Contacts or microphone recordings
  • Secret browser history or activity from other apps

If a user turns personalization off or asks to delete their profile, the dashboard and API include tools for that.

🛠️ Common Problems

Stats stay locked after clicking Load Stats
Make sure you entered the correct admin key. It is case-sensitive and must match the ADMIN_API_KEY environment variable on Render.
API chip shows Offline
Check that the API endpoint is exactly https://faith-content-personalization-engine.onrender.com. Render free-tier services can take 30–60 seconds to wake up after being idle.
Events are rejected
The user may not have accepted consent, or the source name may not be in the approved source list.
Another website cannot send events
Add that website's domain to Render's CORS_ORIGIN environment variable, then redeploy the backend.

💡 Simple Mental Model

Think of it like this:

Dashboard = Control Room You see stats, manage sources, and test things from here.
Render API = Worker Receives signals from apps, builds profiles, runs retention.
Supabase = Filing Cabinet Stores all users, events, consent, and profile data.
