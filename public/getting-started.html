What This Website Is

This website is an admin dashboard for a faith-content personalization engine. It helps you see which approved apps or websites are sending user activity, then turns that activity into anonymous profiles and content recommendations.

The short version:

Vercel dashboard The website you are looking at.
Render API The backend that receives tracking events.
Supabase database Where users, consent, and events are stored.

Your Live URLs

These are the two main links in this setup:

Dashboard:
https://faith-content-personalization-engin.vercel.app

Backend API:
https://faith-content-personalization-engine.onrender.com

The dashboard should use the backend API URL in the API Endpoint field.

How To Use The Dashboard

  1. Open the dashboard URL.
  2. Make sure API Endpoint is set to your Render backend URL.
  3. Enter your private ADMIN_API_KEY in the Admin Key field.
  4. Click Load Stats.
  5. The locked source cards will show real users, events, and profile summaries.

Do not share your admin key with normal website visitors. It is only for you or trusted admins.

What The Dashboard Numbers Mean

  • Connected Apps: approved sources that are allowed to send activity.
  • Total Users: anonymous user profiles created by your app or tests.
  • Total Events: tracked actions like viewing content, reading articles, or watching videos.
  • Data Retention: how many days the system keeps event data before removing old events.

How To Test It Without Another Website

  1. Scroll to Developer Tools.
  2. Click Create User.
  3. Click Accept Consent.
  4. Choose an event type, source, and topic.
  5. Click Track Event.
  6. Click View Profile to see the generated profile.
  7. Go back to the top and click Load Stats again.

How To Connect A Real Website

Scroll to Connect Your App or Website. Choose the source for that website, then copy the embed script.

<script src="https://faith-content-personalization-engine.onrender.com/embed.js?source=official_ministry_website"></script>

Paste that script into the website you want to track. The script shows a consent banner first. If the user allows personalization, it can send approved activity to the backend.

What Counts As An Approved Source

An approved source is a website, app, page, or campaign you trust to send activity into this system.

Current examples:

  • healing_school_app
  • healing_streams_registration_page
  • approved_healing_school_campaign
  • official_ministry_website
  • approved_email_campaign

Privacy Rules

This system is designed to track only consented content activity. It should not collect private messages, passwords, bank details, contacts, microphone recordings, secret browser history, or hidden activity from other apps.

If a user turns personalization off or asks to delete their anonymous profile, the dashboard and API include tools for that.

Common Problems

Stats stay locked

Enter the correct admin key and click Load Stats.

API says offline

Check that the API endpoint is exactly https://faith-content-personalization-engine.onrender.com. Render free services can also take a little time to wake up.

Events are rejected

The user may not have accepted consent, or the source name may not be in the approved source list.

Another website cannot send events

Add that website domain to Render's CORS_ORIGIN environment variable, then redeploy the backend.

Simple Mental Model

Think of it like this: the dashboard is your control room, the Render API is the worker receiving signals, and Supabase is the filing cabinet where the records are kept.
